# Objectif n°1 : avoir les deux mains le plus souvent dans une position symétrique
## Constat n°1 : il faut moins se servir de la souris

### Travailler sur les raccourcis clavier :

- Firefox : « ah mais il existe des plugins Vim »
    - installation du plugin [Vimium](https://vimium.github.io/) pour Firefox et acquisition « passive » de certaines manips de vim (oui, normalement, c'est l'inverse…)
    - « oh ça fait longtemps que je veux vraiment m'y mettre, allez je retente vimtutor », ou l'excellent [vim-adventures](https://vim-adventures.com/).
- LibreOffice : « j'ai décidément du mal à pondre un truc rigoureux avec ces logiciels dits WYSIWYG »
    - qu'est ce que je fais avec LibreOffice ? Pondre du texte, idéalement structuré, avec le moins de manipulations possibles, et l'exporter bien souvent en pdf
    - ai-je besoin de ce type d'outil ? Pratiquement pas.
    - apprentissage de Vim, balisage Markdown et $\LaTeX$.
- OS : utilisation majoritaire sous Linux, Windows quasi uniquement pour le jeu, pas autant de besoins de navigation et manipulations de fenêtres (dommage tout de même quand on s'appelle Windows d'être aussi pas bon en gestion de…fenêtres).

### Identifier tous les clics et mouvements inutiles
- Gestion des fenêtres et des bureaux virtuels :
    - [i3wm](https://i3wm.org/) comme gestionnaire de fenêtres sous Linux
        - permet des bureaux virtuels par écran (sous Windows, les bureaux virtuels associent l'ensemble des écrans à chaque bureau, moins pratique pour moi)
        - [Windows PowerToys](https://learn.microsoft.com/fr-fr/windows/powertoys/) avec FancyZones sous Windows (jusqu'à trouver mieux)
